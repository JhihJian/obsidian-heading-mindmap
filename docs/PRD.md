# Obsidian 思维导图插件 PRD（MVP）

## 1. 产品定位

让一篇 Obsidian Markdown 笔记可以用“可编辑思维导图”方式打开。

导图是笔记内容的标题结构视图。用户在上方导图中查看和编辑标题层级，在下方正文区域查看和编辑当前选中节点的 Markdown 正文，标题、正文和结构变更直接写回原 Markdown 文件。

## 2. 核心原则

- Markdown 文件是唯一数据源。
- 标题层级决定导图层级。
- 导图有一个文档根节点，标题节点来自真实 Markdown 标题；文档根节点本身不写回为 Markdown 标题。
- 节点包含标题和正文，但导图节点只显示标题结构。
- 正文在导图下方的独立编辑区域展示和编辑。
- 导图编辑等价于编辑 Markdown 笔记。
- 跨文件 Markdown 节点可以在当前导图中展开目标文件大纲。
- 展开的跨文件大纲作为只读预览使用。

## 3. 内容规则

- 文件名作为文档根节点标题。
- `# 一级标题` 是文档根下的一级标题节点，可以存在多个同级 H1。
- 若笔记没有一级标题，不自动合成 `# 文件名`；现有标题保持原始 Markdown 层级。
- 节点标题来自 Markdown 标题文本。
- 节点正文为该标题下方、下一个标题之前的 Markdown 内容。
- 子标题下的内容属于子节点。
- 文件节点标题显示目标 Markdown 文件名。
- 文件节点双击后，按目标 Markdown 文件标题层级生成只读子导图。
- 可选把正文中的 Markdown 列表项展示为只读虚拟子节点。
- YAML frontmatter 保留在文件顶部，不并入节点正文。
- 第一个标题前的普通正文作为文档根正文展示和编辑，保存时仍写在第一个标题之前。
- 导图状态信息保存到插件数据，不写入 Markdown 正文。

## 4. 界面布局

- 页面采用上下分屏布局。
- 上方为导图区域，只显示标题树和节点类型标识。
- 导图节点应根据标题内容调整宽高，长标题完整换行展示，不用省略号截断。
- 导图节点保持纯标题结构，操作入口由键盘和视图级命令承载。
- 下方为正文区域，展示当前选中节点标题、来源信息，默认以 Obsidian 阅读视图风格渲染 Markdown。
- 正文区域右上角提供阅读/编辑切换按钮；编辑模式使用 Obsidian 源码编辑视图风格编辑当前节点 Markdown 原文。
- 正文区域右上角提供最小化/展开按钮；最小化后正文内容和拖拽条隐藏，只保留当前节点标题栏，让用户专注导图编辑。
- 正文区域应提供足够高度，并使用下方面板可用宽度展示和编辑内容，支持直接查看和编辑较长正文内容。
- 导图工具栏应提供放大、缩小、适配当前画布查看全图、重置到 100% 缩放的触控按钮。
- 选中导图节点后，下方正文区域同步切换到该节点正文。
- 展开的跨文件大纲节点选中后，正文区域显示只读状态和来源提示。

## 5. 查看能力

- 导图节点显示标题结构。
- 双击普通节点折叠或展开该节点子树。
- 双击文件节点展开或收起目标 Markdown 文件大纲。
- 支持通过视图级开关，把正文列表项作为只读虚拟节点展示或隐藏。
- 导图自动布局。
- 支持基础平移、滚轮缩放和工具栏缩放；移动端可一键适配全图，减少反复横向/纵向滚动。
- 子树折叠状态、文件展开状态、正文区域大小/最小化状态和视野状态应被记忆。
- 页面滚动到下方后，选择、双击或编辑节点时应保持当前视野稳定。

## 6. 编辑能力

- 编辑节点标题，标题编辑在导图节点内联完成。
- 标题编辑采用当前节点内联输入。
- 编辑节点正文，正文编辑在下方正文区域完成，可在阅读视图和编辑视图之间切换。
- 正文编辑模式支持粘贴剪贴板图片；图片应保存为 Obsidian 附件，并在当前光标位置插入图片嵌入链接。
- 新建子节点。
- 新建兄弟节点。
- H1 支持新建同级 H1；只有文档根节点不能新建同级节点。
- 删除节点及其子树。
- 同级节点上下排序。
- 节点升级。
- 从当前库选择 Markdown 文件并添加为文件节点。
- 操作结果立即写回 Markdown 文件。
- 标题最多六级，超过六级的操作给出提示并停止。
- 文档根节点标题由当前文件名决定，不能作为普通 Markdown 标题编辑、删除、排序或升级。
- 展开的跨文件大纲节点为只读，结构、标题和正文编辑操作应给出提示并停止。

## 7. 键盘交互

- `↑ / ↓`：移动到上一个/下一个同级节点。
- `←`：移动到父节点。
- `→`：移动到第一个子节点。
- `Enter`：内联编辑标题。
- `Ctrl + Enter`：聚焦下方正文编辑区。
- `Tab`：新建子节点。
- `Shift + Enter`：新建兄弟节点。
- `Shift + Tab`：节点升级。
- `Alt + ↑ / Alt + ↓`：同级排序。
- `Space`：折叠/展开当前节点子树。
- `Delete`：删除节点。

## 8. 一致性要求

- 导图视图和普通 Markdown 视图内容始终一致。
- 普通视图修改后，导图视图自动刷新。
- 导图视图修改后，普通视图看到的是整洁 Markdown。
- 同一笔记可同时打开导图视图和源码视图。
- 同一文件的多个导图视图保持各自的选中、滚动和缩放状态。
- 插件内部保存子树折叠、文件展开、正文区域大小/最小化和视野状态。

## 9. MVP 边界

- 拖拽节点调整结构放到后续版本。
- 列表项编辑为真实标题节点放到后续版本；本版本只支持可选只读展示。
- 节点颜色、图标、形状自定义放到后续版本。
- 自由摆放节点放到后续版本。
- 导出图片放到后续版本。
- 聚焦模式放到后续版本。
- 完整撤销/重做集成放到后续版本。

## 10. 验收标准

- 普通 Markdown 笔记可用导图方式打开。
- 标题层级与导图结构一致。
- 多个 H1 在文档根下作为同级一级标题展示和编辑。
- 上方导图只显示标题树，节点保持纯标题结构。
- 下方正文区域默认按 Obsidian 阅读视图风格渲染 Markdown，并能通过右上角按钮切换到源码编辑视图风格编辑当前选中节点的 Markdown 原文。
- 下方正文源码编辑视图粘贴图片后，Markdown 中出现 Obsidian 图片嵌入链接，附件文件保存到当前库的附件位置。
- 下方正文区域可通过右上角按钮最小化和恢复，最小化后导图区域获得更多可用空间。
- 正文列表项可选展示为导图虚拟子节点，关闭后不影响 Markdown 内容和真实标题状态。
- 标题可在导图节点内联编辑，编辑过程停留在当前导图上下文。
- 双击普通节点能折叠或展开该节点子树。
- 文件节点双击后能展开或收起目标 Markdown 文件大纲。
- 展开的跨文件大纲节点为只读，编辑操作给出明确提示。
- 工具栏缩放按钮可放大、缩小、重置缩放，并可把当前导图适配到可见画布中。
- 所有键盘操作按定义工作。
- 导图编辑后切回普通视图，Markdown 内容正确、整洁。
- 第六级节点继续新建子节点时给出明确提示。
- 关闭并重新打开后，子树折叠状态、文件展开状态、正文区域最小化状态和视野状态保留。
- 页面滚动到下方后，双击或操作节点时界面不闪烁，滚动位置保持稳定。
